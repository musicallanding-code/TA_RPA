# 06・新任人資經理評估與 90 天計畫

> 基於 2026/6/3 與 CHRO 訪談紀錄,評估 5/4 到職的人資經理。樣本有限(一場會議、到職一個月)。

## 整體判斷
**高潛力、可塑;但目前高度未達 manager 標準——是發展題,不是淘汰題。**

| 強項 | 缺口 |
|---|---|
| 主動性(一個月推 5 案) | 高度:停在「優化」未達「Initiative/無中生有」 |
| 跨域(HR×數據×產品×AI) | 產品嚴謹度:不會代入角色 run scenario、漏 edge case |
| 高度可教練、誠實(扣身心靈一致) | 同理心/風險鈍感:轉嫁痛點、漏看隱私風險 |

價值觀對照:自我超越(成長性)強、身心靈一致(誠實)強,**貢獻價值(Initiative)待長**。

## ⚠️ 重要修正(後續討論的結論)
- **負載過高是「我們派工的問題」**,不是他的失格——5 案橫跨 HR/資料/產品/AI 是 2–3 人守備範圍。
- 高標準 ≠ 高負載 ≠ 提早審判。評估看**成長斜率**,不是滿月就要 senior 成品。
- manager 級合理 ramp 為 3–6 個月。

## 90 天發展與驗證計畫
| 期程 | 重點 |
|---|---|
| 0–30 天 | 期望校準(KPI 從「做了幾案」改「開創/解了誰的痛」);**先收斂 5 案為「主推2＋維持1＋延後2」**;指派 CTO/CDO 當技術導師教「代入角色 run scenario」 |
| 30–60 天 | 給開放命題驗證 Initiative;觀察「轉嫁痛點」是否改用 AI-First「雙方都不痛」 |
| 60–90 天 | 三缺口是否可見改善;能否在沒人餵的情況下產出戰略洞察 → go/no-go |

## 帶人權
在「轉嫁痛點/同理心」明確改善前,暫緩賦予管理他人的職權。

## 給管理者的提醒
最大資產=誠實＋可教練;最大風險=停在優化層、把痛丟給別人。**別把「態度好」誤當「能力到位」而過早授權帶人。**
