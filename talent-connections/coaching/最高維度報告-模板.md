# 最高維度報告・模板(讓全經營層買單)

> 用法:這不是「報 5 件事」,是「**證明一件事,讓八個長各自看見自己的贏**」。
> 鐵則:**每一句高維度,底下都要壓一個落地證據。** 證據三角沒到位,就退回務實版,別講最高維度。

---

## ① 主命題(一句,先寫這句)
> 我證明了:**______________________________**
> (範例:數據＋AI 能把 HR 從後勤變成替 CMoney 創造競爭力的引擎,而且這套模式全公司可複製。)

## ② 證據三角(高維度不淪為 PPT 的關鍵——三者缺一不可)
| 證據 | 內容(填實) | 檢核 |
|---|---|---|
| **A. 改變決策的數字** | 內推 cost per quality hire 比其他管道【低 __%】/ 轉換率【__%】最高 → 建議重配【__】預算 | ☐ 是「能改決策」的數字,不是描述性 |
| **B. 已交付(可 demo)** | 座位分機系統【已上線/POC 驗收】;MCP×SQL 儀表板【雛形】 | ☐ 現場開得出來、不是截圖 |
| **C. 可複製模式** | MCP×SQL×Agent 的 playbook,可搬到【__部門】 | ☐ 別人抄得走,不是只有我會 |

## ③ 對公司的意義(把證據接到戰略)
- 人才密度:______(質量加權進用＋內推留任如何提升)
- AI-First 轉型:______(HR 當試點 → 全公司燈塔)
- 雇主品牌:______(人才樞紐內容對外、降 CAC)
- 組織效能與治理:______(省下工時/成本、內控可稽核)

## ④ 下一步賭注＋需要的支持
- 賭注:人才樞紐 MVP(合規版:opt-in 人才庫＋自家管道整合,不爬履歷)
- 需要拍板:______
- 需要擋掉的阻力:______

## ⑤ 合規與架構(讓 CIO/CFO 安心)
- 資料治理/PDPA:______ ・ build-vs-buy:底層用現成、只自建內容層

## 買單矩陣(寫每段時自問:這位長的贏在哪?)
| 長 | 他要看到 |
|---|---|
| CFO | 改變決策的數字 |
| CDO | 可複製的 AI-First 範本 |
| CMO | 對外雇主品牌、降 CAC |
| CTO | 建得對、可複用、不重造 |
| CIO | 資料治理/PDPA 內建 |
| COO | 可量化的工時釋放＋座位上線 |
| CHRO | 人才密度↑＋價值觀對齊 |
| 幕僚長/CEO | 接公司戰略:人才密度=護城河 |

## 🚫 三個紅旗(踩到就退回務實版)
1. **討好型失焦**:給八個承諾。→ 只證明一個命題。
2. **空中樓閣**:高維度卻沒有證據三角。→ 先補證據。
3. **過早喊大**:「全公司可複製」還沒第二個案例驗證。→ 先小試點再宣稱。
