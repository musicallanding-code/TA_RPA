# 月報 AI 自動化・架構(HR 先行)

> 決策(2026-06,Leo 拍板):**全自動**——Agent 排程自動拉數據、寫入 Google Doc,不需人工。
> 架構:**Skill 定標準 ＋ Agent 執行 ＋ git 管理 ＋ HR 試點**(對齊 CMoney AI Brain)。

## 為什麼是「Skill ＋ Agent」而非二選一
- **Skill**=月報的「標準/知識」(OKR 結構、資料來源、去識別規則),放 **git**,版本可追溯,所有主管共用 → 對應 AI Brain 的 KNOWLEDGE/SKILL 支柱。
- **Agent**=執行引擎,透過 **MCP** 拉數據、套 Skill、寫 Google Doc。
- 一句話:**Skill 是大腦的知識,Agent 是手腳的執行。**

## 全自動管線
```
排程(每月) → Agent
  ① MCP 拉數據：招募系統/JIRA · 104 · 人力異動 DB · 部落格/活動 · 推薦數
  ② 套用 Skill：hr-monthly-report(OKR/KR1-3/人力異動/AI 場景)
  ③ 自動套用「去識別 + 權限」規則於敏感段落
  ④ Google Docs MCP 寫入：新增當月「# YYYYMMDD」區塊（不覆蓋歷史）
  ⑤ Fail-safe + audit log + 完成通知(PM/HR)
```

## 護欄(讓全自動不出事)
1. **去識別自動化**:離職名單/人才盤點/壞的離職 → 去識別或寫入權限受限區。
2. **Doc 權限控管**:自動寫入的 Doc 僅限主管/HR,非公開。
3. **Fail-safe**:資料缺漏/異常 → 標「待補/異常」、不杜撰、發警示。
4. **可追溯**:Google Doc 版本歷史 ＋ git audit log。
5. **事後抽查通知**(非阻斷):寫完通知 PM/HR,可回溯修正。

## 落地階段
1. **建置**:Skill(已建)＋ MCP 連接器(資料源＋Google Docs)＋去識別/權限規則＋排程。
2. **影子跑 1 期**(建議):首期自動產出先寫草稿 Doc,確認連接器與數據正確。
3. **全自動**:確認後轉排程自動寫正式 Doc。

## 殘餘風險(全自動相對半自動多承擔)
敏感人事資料**無人即時審核**即發布;護欄把風險壓低但非零,異常情境可能短暫露出不該露的內容(有版本歷史可回溯)。此為「全自動」的取捨,已由決策者選擇承擔。

## 角色分工
| 角色 | 負責 |
|---|---|
| PM | 維護 Skill(月報標準)、去識別規則 |
| Agent | 排程執行:拉數據、產稿、寫 Doc |
| HR 主管 | 事後抽查、處理警示 |
| 工程/IT | 建 MCP 連接器、排程、權限 |

## 後續可複製
HR 跑通後,同一「Skill＋Agent」模式可複製到其他部門月報 → 全公司月報 AI 自動化的燈塔。

# POC 時程評估(14 工作天 POC / 7 天驗收 / 月底上線)

> 圓桌結論:**可行,但有條件**——須 scope 收窄＋權限 Day 1 啟動;「全資料源＋全自動」一次到位則一個月不現實。
> POC 採**半自動(人比對)**驗證,上線後再轉全自動(不違背全自動終局,只是分階段)。

## 14 工作天 POC 拆解
| 期間 | 工作 | 負責 |
|---|---|---|
| D1–2 | 鎖定 POC 範圍(選 1–2 個**已可存取**資料源,如人力異動 DB＋招募系統)、**立即申請所有權限**(資料源＋Google Docs 寫入)、定去識別規則 | Leo(架構)＋IT |
| D3–7 | 建選定資料源的 MCP 連接器、串 `hr-monthly-report` Skill 產草稿 | 工程 |
| D8–11 | 去識別/權限套用、寫入草稿 Google Doc、fail-safe | 工程＋CIO |
| D12–14 | 自測、修正、產出「自動生成一期月報草稿」POC demo | 工程＋Leo |
| 驗收 D15–21(7天) | 艷林/運清＋HR 主管比對草稿 vs 真實數據、修正去識別與格式 | COO＋HR |
| 上線(月底) | 範圍內 MVP 上線;先半自動(人抽查),穩定後轉全自動排程 | Leo＋工程 |

## 時程風險(依嚴重度)
1. **🔴 權限審批(頭號殺手)**:人力異動 DB／104／Google Workspace 寫入權,常以「週」計。**D1 不啟動 → 月底上線跳票。**
2. **🟡 資料源就緒**:部分資料源「建置中」未就緒 → POC 只選「已可查詢」的資料源。
3. **🟡 工程人力**:未鎖定 owner 與工時,時程即空談。

## 成立前提
① 工程人力鎖定;② POC scope 收窄(1–2 資料源);③ POC 半自動驗證、上線再全自動;④ 權限 D1 送出申請。
