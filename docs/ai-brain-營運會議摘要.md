# AI Brain 營運會議摘要與兩層組織架構

> 整理：供 `/cmoney-board` 八長圓桌討論使用

## 一、簡報重點摘要

兩條主線交織——一條是「2B 數據產品怎麼賣」，一條是「用 AI Brain 加速組織」。

### 主線 A：把「發票/消費數據」產品化成 2B 訂閱
- **客戶分類創新**：不按產業分，按「**付錢的真實理由**」分三種角色——
  - **決策的眼睛**（看市場做決策，如王品、凱基銀行）
  - **提案武器**（拿去贏單搶預算，如 LiTV、陽獅）
  - **市場研究原料**（自己加工，如 Nielsen、Ipsos）
- **一份資料，三種加工，三層價值**（核心商模設計）：
  - 發票數據 → ① 市場洞察（看見市場）→ ② 標籤服務（看清自家人）→ ③ 受眾包+CRM（採取行動）
  - 客戶介入越深 → **訂閱黏性越高**
- **Q3 OKR**：消費資料庫正確率 **40%→90%**、新商模驗證 **1 天交付**、累積營收目標 **4,194 萬（YoY +67%）**
- **兩大瓶頸已有解方**：資料正確性（靠 MCP 隨時補各庫）、驗證周期過長（MCP 即時數據 × Vibe Coding，交付從「一次」縮到「一週」）

### 主線 B：AI Brain＝可被 Claude 調用的「組織大腦」
- **目標**：打造 **AI Native 組織**——「Vibe Coding 做出產品很容易，做對事情才難；AI Brain 是為了確保大家用對方法、解決真正存在的問題」。
- **用 git 管理、Claude 可直接調用**，三大支柱：
  - **KNOWLEDGE 知識**：團隊目標、名詞定義、內部指標 → 讓 AI 有依據不亂答
  - **DATA 數據**：Source code、Event data、假設驗證表 → 驗證時要翻的真實底料
  - **SKILL 能力**：把最佳做法封裝成「隨叫隨到的 AI 同事」（工程架構師、設計審查員、簡報顧問、風險評估師）
- **終局想像**：當「每個人的 agent」也接進來，AI Brain 進化成「**人＋AI 分身混編的組織**」，戰力脫離「人在不在」的限制。

### 現況與卡點
| ✅ 正向訊號 | ⚠️ 問題 |
|---|---|
| 資料調用速度快，可行性已驗證 | **PRD skill 反而拖慢**：問太多、規劃過重、沒 MVP 概念 |
| PM/UI 能產出合規範的 vibe coding | **figma MCP 串接成效不彰**：設計師手動調比 AI 快 |
| 工程端已開始實際試用 AI Brain | **歷史資料不夠**：AI 不知道團隊目標與過往，決策空泛 |

**最大誠實點**：「現在能回答問題、做好事，但最大價值還沒展示出來——卡在**資料源還在建置中**。」

## 二、兩層 AI 組織架構

### 架構層 1：MCP 整合層 =「組織的神經系統」
> 透過 MCP 將所有資料、系統整合成能讓 AI/LLM 解讀的平台

```
        ┌─────────── AI / LLM (Claude) ───────────┐
        │            統一查詢介面 (MCP)            │
        └──┬─────┬─────┬─────┬─────┬─────┬────────┘
           │     │     │     │     │     │
        資料庫  JIRA  GWS  發票    figma  程式碼
        (消費)  (任務) (信件) rawdata (設計) (source)
```
- **本質**：把分散的系統「**可被即時 query**」——重點不是搬資料，是讓 AI 能在當下調用真實資料。
- **價值**：消除「等資深的人、層層轉手」的阻力，任何人都能自助驗證假設。
- **風險紅線**：發票/消費 rawdata 涉 **個資法去識別**；30 億筆數據變現邊界；MCP 把內網全打通 = **資安權限治理**。

### 架構層 2：AI Brain =「組織的認知與記憶」
> 餵養組織的歷程、歷史、文化後，成為 AI Brain，夥伴透過它快速上手

```
            AI BRAIN (git 管理、版本可追溯)
   ┌────────────┬────────────┬──────────────┐
   │ KNOWLEDGE  │    DATA     │     SKILL     │
   │  共識/事實  │  原始素材    │ 隨叫隨到 AI 同事│
   │ 目標·名詞·  │ code·event· │ 架構師·審查員· │
   │ 指標定義    │ 假設驗證表   │ 顧問·風險官    │
   └────────────┴────────────┴──────────────┘
            ↓ 終局：人 + AI 分身混編組織
```
- **本質**：架構層 1 解決「**調得到資料**」；架構層 2 解決「**知道怎麼用、為什麼用**」——它是組織的**記憶與判斷力**。
- **HR/組織意涵**：新人上手曲線、知識不隨人離職流失、SKILL 把資深者判斷封裝成可複用資產 → 牽動**人才密度**與**組織能力沉澱**。
- **關鍵前提**：架構層 2 品質 = 餵進去的「歷程/歷史/文化」品質。目前卡在「資料源還在建置中」——這是資料治理＋知識管理工程，不是裝工具就好。

**兩層關係一句話**：MCP 是神經（連得到、傳得快），AI Brain 是大腦（記得住、判斷對）。先有神經才有大腦，但大腦的價值遠大於神經——CMoney 目前卡在「神經通了、大腦還在發育」。
