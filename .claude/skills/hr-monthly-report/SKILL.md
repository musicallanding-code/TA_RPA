# CMoney HR 月報・產生標準

> 用途:把每月 HR 數據彙整成固定結構的月報,新增一個 `# YYYYMMDD` 區塊到 Google Doc。
> 此 Skill 定義「報告長什麼樣、資料從哪來、敏感資料怎麼處理」;由 Agent(MCP 拉數據)或主管手動套用。

## 一、固定結構(每月一個日期區塊)

```
# YYYYMMDD

# OKRs
## O：幫助專案與部門將人才補齊

# KR1：擴大招募管道
## 一、招募活動類      （MA Program、海外數據 Advisor/BD、校招…）
## 二、獵頭            （開缺職務、Hunter 推薦數、推薦數月比較）

# KR2：強化雇主品牌形象與影響力
## 一、雇主品牌活動    （C Talk+、104 職涯博覽會、校園招募）
## 二、部落格/內容     （現況、交稿狀況）

# KR3：優化人資與招募流程，提升人才管理與招募效能
## 一、招募現況        （開缺/補缺數據、進行中件數）
## 二、各部門 P0、P1 招募情形（工程/行銷/業務/產品/數據/營運/特殊職務）
## 三、HR 系統與 AI 應用場景

# 人力異動報告
## 一、近 12 個月每月到離職人數(率)及淨人數變化
## 二、當月人力異動概況
## 三、人才盤點資訊（含離職狀態：好的/壞的離職—優秀人才流失）
```

## 二、資料來源對應(Agent 拉數據用)
| 區塊 | 資料來源 |
|---|---|
| KR1 招募活動/獵頭/推薦數 | 招募系統、JIRA、Hunter 數據 |
| KR2 雇主品牌/部落格 | 活動排程、部落格交稿表 |
| KR3 開缺/補缺、各部門 P0/P1 | 招募系統、缺額表、JIRA |
| 人力異動、到離職率、淨變化 | 人力異動 DB / HR 系統 |
| 人才盤點、離職狀態 | 人才盤點表 |

## 三、敏感資料規則(全自動下強制)
- **離職名單、人才盤點、「壞的離職/優秀人才流失」**屬敏感人事資料:
  - 依設定**去識別**(以代號/職能代稱),或寫入**權限受限區**。
  - 寫入的 Google Doc **僅限主管/HR 權限可見**,不得公開。
- 資料缺漏或異常 → 該段標記「待補/異常」,**不杜撰數字**,並觸發警示。
- 數字一律引用真實來源,標明統計區間;留存/趨勢類數據註明觀察期。

## 四、格式與語氣
- 繁體中文、條列、客觀;OKR/KR 階層清楚。
- 沿用既有 Doc 的「# 日期」分段方式,**新增當月區塊、不覆蓋歷史**。
- 數據優先、不空泛;與上月可比的指標標出變化(如「推薦數 11→20」)。

## 五、產出
輸出整段 Markdown(可直接寫入 Google Doc)。Agent 模式下,經 Google Docs MCP 寫入指定文件;手動模式下,主管複製貼上。
